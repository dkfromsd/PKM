## 1. 분야별 API 종류 및 사용법

- **웹 (Web):** 주로 **REST API**나 **GraphQL**을 사용합니다. HTTP 프로토콜을 통해 데이터를 주고받으며, 브라우저와 서버 간의 통신에 필수적입니다.
- **임베디드 (Embedded):** 하드웨어를 직접 제어하는 **HAL(Hardware Abstraction Layer) API**나 **C 표준 라이브러리**를 사용합니다. 센서 값을 읽거나 모터를 제어할 때 씁니다.
- **빅데이터:** **REST API** 외에도 대량의 데이터를 스트리밍하기 위한 **gRPC**나 각 프레임워크(Spark, Kafka) 전용 API를 사용합니다.
- **사물인터넷 (IoT):** 전력 소모가 적은 **MQTT**나 **CoAP** 프로토콜 기반 API를 사용해 기기 간 상태를 공유합니다.
- **물리 인공지능 (Robotics/AI):** 로봇 운영체제인 **ROS API**나 **TensorFlow/PyTorch API**를 통해 실시간 데이터 처리와 추론을 수행합니다. 

## 2. DevOps 핵심 도구의 이유

- **Go:** 이 모든 도구(Docker, K8s, Terraform)의 모태가 되는 언어입니다. 컴파일 속도가 빠르고 동시성 처리가 뛰어나 인프라 도구 제작에 최적입니다.
- **Docker:** 애플리케이션을 환경에 구애받지 않게 포장(컨테이너화)하여 "내 컴에선 되는데?"라는 문제를 해결합니다.
- **Terraform:** 코드로 인프라를 관리(IaC)하게 해줍니다. 클릭 대신 스크립트로 서버 수백 대를 동시에 세팅합니다.
- **Prometheus:** 시스템의 상태(CPU, 메모리, 트래픽)를 시계열 데이터로 수집하고 감시하는 '눈' 역할을 합니다. 

## 3. 언어별 API 활용 예시 및 융합

이 기술들은 **'Go로 만든 도구(Docker/Terraform) 위에 각 언어로 짠 앱을 올리고, Prometheus로 감시하는'** 방식으로 융합됩니다. 

| 언어               | 주요 역할       | API 예시                                             |
| ---------------- | ----------- | -------------------------------------------------- |
| **TypeScript**   | 웹 프론트엔드/백엔드 | **Fetch API**를 사용하여 서버에서 데이터를 가져와 화면에 렌더링.         |
| **Go**           | 고성능 마이크로서비스 | **gRPC API**를 구현하여 서비스 간에 초고속으로 데이터를 교환.           |
| **Python**       | AI 모델 배포    | **FastAPI**를 통해 딥러닝 모델의 추론 결과를 웹 API 형태로 제공.       |
| **Java**         | 대규모 기업용 시스템 | **Spring Boot**의 REST API를 사용하여 안정적인 데이터 처리 서버 구축. |
| **JS (Node.js)** | 실시간 통신      | **Socket.io API**를 사용하여 IoT 기기와의 실시간 양방향 통신 구현.    |

**융합 시나리오:**

1. **Terraform**으로 클라우드 서버를 할당합니다.
2. **Go**로 짠 백엔드와 **Python** AI 모듈을 **Docker**로 말아서 올립니다.
3. 서비스 간 데이터는 **REST/gRPC API**로 소통합니다.
4. 이 과정에서 발생하는 모든 지표는 **Prometheus**가 수집해 대시보드에 뿌려줍니다
