 Terraform 코드와 시나리오를 바탕으로, **개인용 비서 에이전트(AI Agent)**를 **클라우드(AWS)와 K8s 환경**에 구축할 때 각 기술이 어떻게 유기적으로 연결되는지 설명해 드리겠습니다.

1. 전체 아키텍처 흐름 (Architecture Flow)

제시하신 S3 → SQS → Processor 구조를 인공지능 비서 서비스에 대입하면 다음과 같습니다.

1. **S3 (데이터 보관):** 사용자의 음성 파일이나 텍스트 로그를 보관합니다.
2. **SQS (작업 대기열):** 새로운 데이터가 들어오면 처리하라는 "신호"를 보관합니다.
3. **Processor (K8s 상의 에이전트):** SQS에서 신호를 받아 AI 모델(Python 등)을 돌려 응답을 생성합니다.
4. **Prometheus (모니터링):** 이 전체 과정이 얼마나 빠른지, 에러는 없는지 감시합니다.

---

2. 기술별 역할 및 연결 방식

① Terraform (인프라의 뼈대)

제시하신 `main.tf`와 `modules` 구조를 사용하여 AWS의 S3, SQS, 그리고 K8s 클러스터(EKS) 자체를 생성합니다.

- **연결점:** Terraform이 생성한 SQS의 **Queue URL**은 환경 변수 형태로 K8s에 떠 있는 애플리케이션(Go/Python)에 전달됩니다.

② Docker (애플리케이션의 포장)

비서 에이전트가 사용하는 라이브러리(PyTorch, OpenAI SDK 등)를 환경에 상관없이 돌아가도록 패키징합니다.

- **연결점:** `Dockerfile`로 빌드된 이미지는 클라우드 저장소(ECR)에 저장되고, K8s가 이를 당겨와서 실행합니다.

③ Kubernetes (K8s - 운영의 자동화)

Terraform으로 만든 서버 위에서 Docker 컨테이너를 관리합니다.

- **연결점:** 비서 에이전트에게 요청이 몰리면 **Auto-scaling**을 통해 에이전트 숫자를 자동으로 늘립니다.

5. **Terraform:** "AWS에 S3 버킷과 SQS 큐를 만들고, 에이전트가 돌아갈 K8s 클러스터를 세팅해!" (코드 실행)
6. **User:** 스마트폰 앱(JS)으로 음성 명령을 내립니다. 음성 파일은 S3에 저장됩니다.
7. **S3 → SQS:** 파일이 저장되면 SQS에 "새로운 작업 도착" 메시지가 생성됩니다.
8. **Go (K8s Pod):** SQS API를 감시하다가 메시지를 낚아채서 **Python(AI Pod)**에게 "이것 좀 분석해줘"라고 gRPC API로 요청합니다.
9. **Python:** AI 모델을 돌려 답변을 만들고, 결과를 DB에 저장합니다.
10. **Prometheus:** 이 모든 과정(S3 저장부터 답변까지)이 몇 초 걸렸는지 그래프로 보여줍니다.

1. Python 에이전트 코드 (Processor 역할)

이 코드는 SQS API를 통해 메시지를 가져오고, 처리 결과를 로그로 남깁니다. (실제 환경에서는 여기에 OpenAI 같은 AI API를 연결합니다.)

python

```
import boto3
import time
import os

# Terraform이 생성한 SQS URL을 환경변수로 받음
sqs_url = os.getenv("SQS_QUEUE_URL")
sqs = boto3.client('sqs', region_name='us-west-2')

def process_ai_task(message_body):
    print(f"AI 비서 작업 시작: {message_body}")
    # 여기에 AI 모델 추론 로직(예: LLM 호출)이 들어갑니다.
    time.sleep(1) 
    return "작업 완료!"

while True:
    # 1. SQS API로 메시지 수신 (Long Polling)
    response = sqs.receive_message(
        QueueUrl=sqs_url,
        MaxNumberOfMessages=1,
        WaitTimeSeconds=20
    )

    if 'Messages' in response:
        for msg in response['Messages']:
            # 2. 비즈니스 로직 실행
            result = process_ai_task(msg['Body'])
            
            # 3. 처리가 끝난 메시지는 삭제 API 호출
            sqs.delete_message(
                QueueUrl=sqs_url,
                ReceiptHandle=msg['ReceiptHandle']
            )
            print(f"처리 결과: {result}")
```

Use code with caution.

2. Kubernetes 배포 설정 (YAML)

위에서 만든 Python 앱을 Docker 이미지(`my-ai-agent:v1`)로 만들었다고 가정하고, K8s에 올리는 설정입니다.

- **Prometheus 연결:** `annotations` 부분을 통해 Prometheus가 이 앱의 상태를 자동으로 수집

yaml

```
apiVersion: apps/v1
kind: Deployment
metadata:
  name: ai-agent-deployment
  labels:
    app: ai-agent
spec:
  replicas: 3  # 에이전트 3개를 띄워 부하 분산
  selector:
    matchLabels:
      app: ai-agent
  template:
    metadata:
      labels:
        app: ai-agent
      annotations:
        # Prometheus가 이 Pod의 메트릭을 수집하도록 설정
        prometheus.io/scrape: "true"
        prometheus.io/port: "8080"
    spec:
      containers:
      - name: ai-agent
        image: my-repo/my-ai-agent:v1
        env:
        - name: SQS_QUEUE_URL
          value: "https://amazonaws.com" # Terraform 출력값 입력
        resources:
          requests:
            memory: "512Mi"
            cpu: "500m"
```

3. 기술 간의 '연결 고리' 정리

4. **Terraform → K8s:** Terraform이 SQS를 만들고 그 **URL**을 알려줍니다. 우리는 그 URL을 위 YAML의 `env` 섹션에 넣습니다.
5. **Docker → K8s:** Python 코드를 이미지로 구워 저장소에 올리면, K8s가 `image: my-repo/my-ai-agent:v1` 설정을 보고 컨테이너를 실행합니다.
6. **Python → AWS API:** Python 내부의 `boto3` 라이브러리가 AWS SQS API와 통신하여 데이터를 주고받습니다.
7. **Prometheus → Python:** Python 앱이 8080 포트로 "지금 처리 중인 메시지 수" 같은 데이터를 내보내면, Prometheus API가 이를 정기적으로 긁어(Scrape) 갑니다.

8. 융합의 핵심 (DevOps)

이렇게 구성하면 **Go**로 만든 인프라 도구(Terraform/K8s)가 **Python**으로 만든 AI 로직을 안전하게 감싸서 서비스하게 됩니다. 결과적으로 개발자는 로직(Python)에만 집중하고, 운영은 도구들이 알아서 하게 되는 구조가 완성됩니다.

## 배포 

1. CI/CD: 코드를 K8s에 자동 배포하기 (GitLab 기준)

제시하신 `project/` 구조에 있던 `.gitlab-ci.yml` 파일이 이 역할을 수행합니다.

1. **Build (Docker):** 개발자가 Python 코드를 수정해 Push하면, GitLab Runner가 **Docker API**를 사용해 이미지를 빌드합니다.
2. **Push:** 빌드된 이미지를 AWS ECR(컨테이너 저장소)에 저장합니다.
3. **Deploy (K8s/Terraform):** **Terraform API** 혹은 `kubectl`을 호출하여 K8s 클러스터에 "새 버전 이미지(`v2`)로 교체해!"라고 명령합니다.

**[.gitlab-ci.yml 예시 코드]**

yaml

```
stages:
  - build
  - deploy

docker-build:
  stage: build
  script:
    - docker build -t my-repo/ai-agent:$CI_COMMIT_SHA .
    - docker push my-repo/ai-agent:$CI_COMMIT_SHA

deploy-to-k8s:
  stage: deploy
  script:
    - kubectl set image deployment/ai-agent-deployment ai-agent=my-repo/ai-agent:$CI_COMMIT_SHA
```

Use code with caution.

2. Prometheus: 수집한 데이터 시각화하기 (Grafana)

Prometheus는 숫자 데이터를 수집하고, **Grafana**라는 툴을 API로 연결해 시각화합니다.

1. **Expose:** Python 앱 내부에 `prometheus_client` 라이브러리를 써서 API 엔드포인트(`/metrics`)를 엽니다.
2. **Scrape:** Prometheus 서버가 K8s에 떠 있는 Python Pod들을 돌아다니며 "지금 처리량 어때?"라고 묻고 데이터를 긁어갑니다.
3. **Visualize:** **Grafana** 대시보드에서 Prometheus를 데이터 소스로 등록합니다.
    - **쿼리 예시 (PromQL):** `rate(ai_tasks_completed_total[5m])`
    - **결과:** "최근 5분간 AI 비서가 초당 몇 개의 요청을 처리했는지" 실시간 그래프가 그려집니다.
