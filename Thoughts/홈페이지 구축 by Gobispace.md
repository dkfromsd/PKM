## 🕒 5시간의 디버깅 타임라인

### 1. "함수를 찾을 수 없음" (SDK의 신기루)
*   **시도:** 초기에는 `gobi.vault.search()`와 `gobi.vault.getMetadata()`를 사용하여 폴더를 자동 탐색하려 함.
*   **에러:** `TypeError: gobi.vault.search is not a function`.
*   **통찰:** 현재 사용 환경의 SDK 버전이 해당 고수준 함수들을 지원하지 않음을 파악. 더 원초적인 함수로 접근 방식을 전환해야 했음.

### 2. "업데이트 피드"로의 선회 (절반의 성공)
*   **시도:** `gobi.listBrainUpdates()`로 모든 게시물을 긁어오는 방식 선택.
*   **결과:** **성공!** 데이터가 화면에 나오기 시작함.
*   **문제:** 이 API는 실제 파일 구조가 아닌 '게시물(업데이트)'을 반환함. 따라서 볼트의 체계적인 폴더 구조를 그대로 반영하기에는 한계가 있었음.

### 3. "샌드박스의 감옥" 위기 (404 에러의 순환)
*   **시도:** `gobi.listFiles('')`를 사용하여 실제 폴더 스캔 시도.
*   **현상:** 홈페이지가 위치한 `app/` 폴더 내부만 보이고 외부 폴더는 전혀 보이지 않음.
*   **장벽:** Gobi 홈페이지는 보안상 `app/` 디렉토리에 격리(Sandboxed)되어 있음. `..`이나 `/`로 상위 폴더를 보려 하면 "Permission Denied" 또는 "Failed to Fetch" 에러 발생.

### 4. "터미널 모드"를 통한 돌파구 (경로의 재발견)
*   **전략:** 임시 **터미널 스타일 디버그 대시보드**를 구축하여 모든 경로를 테스트.
*   **방법:** `.`, `/`, `..`, `../..` 그리고 마지막으로 `../Stock`을 차례로 테스트함.
*   **발견:** 결정적인 돌파구! 서버가 상위 폴더(`..`)의 목록을 보여주는 것은 막아두었지만, **이름을 명시하여 형제 폴더에 직접 접근하는 것(`../Stock`)은 허용**한다는 사실을 발견. 샌드박스를 탈출할 열쇠를 찾음.

### 5. "프리미엄 에디토리얼" 디자인 완성
*   **디자인 변화:** 단순한 버튼 나열에서 한 페이지에 모든 카테고리가 펼쳐지는 **스크롤 갤러리** 방식으로 전면 개편.
*   **타이포그래피:** 기본 폰트를 버리고 우아한 `Cormorant Garamond`와 현대적인 `Plus Jakarta Sans`를 조합.
*   **기능 추가:** 비밀 로그창(L 키 토글)과 스트리밍 방식의 AI 셰프 채팅 인터페이스 구현.

### 6. `nav=false`의 변수와 다중 경로 파상 공세 (마지막 퍼즐)
*   **새로운 시도:** Gobi 자체 네비게이션을 숨기기 위해 `homepage: ./app/index.html?nav=false` 설정 적용.
*   **돌발 상황:** 네비게이션은 숨겨졌으나, 잘 나오던 주방의 내용이 다시 모두 사라짐.
*   **원인 분석:** `?nav=false` 파라미터가 붙으면 Gobi 서버가 페이지의 **'뿌리 주소(Root Path)'**나 샌드박스 정책을 변경하여 기존의 `../Stock` 경로를 무효화함을 발견.
*   **최종 솔루션:** **"다중 경로 파상 공세(Multi-Path Probe)"** 로직 도입. `../cat`, `./cat`, `/cat`, `cat` 등 모든 가능성 있는 경로를 순차적으로 찔러보고 가장 먼저 응답이 오는 경로를 자동으로 선택하게 함으로써 환경 변화에 완벽히 대응하는 '전천후 엔진' 완성.

## 🛠️ 핵심 기술적 발견 요약

| 기능 | 실패한 시도 | 성공한 솔루션 |
| :--- | :--- | :--- |
| **폴더 탐색** | `gobi.vault.search()` | 미리 정의된 `categories` 리스트 + `listFiles()` |
| **경로 지정** | `listFiles('/')` | **다중 경로 Probe** (`../`, `./`, `/` 등 자동 탐색) |
| **데이터 처리** | 객체에 `forEach` 사용 | 배열 변환을 위한 강력한 `extract()` 헬퍼 도입 |
| **렌더링** | 기본 마크다운 출력 | `marked.js` + 프리미엄 에디토리얼 스타일링 |
| **로딩 안정성** | 즉시 API 호출 | SDK 주입을 기다리는 **30회(6초)** 재시도 루프 |
| **환경 대응** | `?nav=false` 시 404 | 유연한 경로 탐색 로직으로 어떤 파라미터든 대응 |
