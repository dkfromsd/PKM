
# 📔 Mickey's Kitchen: 진화의 기록 (v3.2 - v3.8)
### Gobi SDK 기반 커스텀 홈페이지 업데이트 히스토리

**날짜:** 2026-04-14
**작성자:** Mickey Lee (Head Chef)

Gobi 플랫폼의 독특한 클라우드 샌드박스 환경과 `nav=true/false` 설정에 따른 경로 가변성을 극복하며 "Mickey's Kitchen"이 완성되어간 기술적 여정을 기록합니다.

---

## 🚀 버전별 주요 업데이트 내역

### 1. v3.2 - Multi-Path Probe Engine
*   **배경:** `homepage: ./app/index.html?nav=false` 설정 시 Gobi 서버가 현재 페이지의 기준 경로(Base Path)를 변경하여 기존의 `../Stock` 경로가 404 에러를 내며 모든 내용이 사라짐.
*   **해결책:** **"다중 경로 파상 공세"** 로직 도입.
*   **핵심 기술:** `./`, `../`, `/`, `Vault/` 등 4가지 이상의 후보 경로를 정의하고, `listFiles` API가 유효한 응답을 줄 때까지 순차적으로 테스트하여 환경 변화에 자동 대응함.

### 2. v3.4 - Self-Healing Engine
*   **배경:** 특정 환경에서 Gobi SDK가 `vaultId`를 놓치는 현상 발생. `SDK Connected. Vault: undefined` 에러로 인해 API 요청 주소가 깨지는 문제(`.../api/v1/tree/Stock`) 발견.
*   **해결책:** **"자가 치유(Self-Healing)"** 로직 도입.
*   **핵심 기술:** SDK가 ID를 찾지 못할 경우, 브라우저 주소창(`@mickeyfromsd`)에서 슬러그를 직접 추출하여 시스템에 강제로 주입함. 이를 통해 API 요청 주소를 정상적으로 복구함.

### 3. v3.6 - Bulletproof Engine
*   **배경:** 목록(List)은 정상적으로 노출되나, 레시피 카드 클릭 시 세부 내용을 읽어오지 못하는 `ERROR: Failed to read` 현상 발생. 목록용 경로와 파일 읽기용 경로 기준이 서로 다름을 확인.
*   **해결책:** **"읽기 전용 다중 경로 Probe"** 적용.
*   **핵심 기술:** `openRecipe` 함수 내부에 별도의 다중 경로 탐색 루프를 주입. 파일 위치가 어디에 있든(상위 폴더 혹은 같은 폴더) 반드시 찾아내어 읽어내도록 설계함.

### 4. v3.8 - Precision Engine (최종 안정화 버전)
*   **배경:** 상단 카테고리나 `TOP` 버튼 클릭 시 화면이 검게 변하는 현상과 서버 응답이 문자열이 아닌 객체(Object)로 올 때 렌더링이 실패하는 문제 해결.
*   **해결책:** **"비침투형 SPA 네비게이션"** 및 **"데이터 형식 자동 교정"** 도입.
*   **핵심 기술:**
    *   `<a>` 태그의 해시 링크(`href="#"`)가 Gobi 플랫폼의 라우팅과 충돌함을 인지, 모든 링크를 `<button>`과 `scrollIntoView`로 교체하여 페이지 이동 오인을 원천 차단.
    *   서버 응답이 `{data: "..."}` 혹은 `{content: "..."}` 형태일 경우를 대비해 데이터를 정밀하게 추출하는 전처리 로직 강화.
    *   모달 배경색을 불투명하게 고정하여 Gobi 배경색과의 충돌 해결.
